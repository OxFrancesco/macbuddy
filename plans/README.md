## Findings considered and rejected

Recorded so nobody re-audits them. Each was checked against the live code.

- **`sources[0]` crash in single-icon regeneration** — false: `guard
  !sources.isEmpty else { return }` (DockPaletteModel.swift:186) dominates the
  access at line 206.
- **Data races on `aiResults`/`generatingPaths`/model state from Tasks** —
  false: the project builds with `SWIFT_DEFAULT_ACTOR_ISOLATION: MainActor`;
  the model and all its mutation sites (including the task-group result loops)
  are MainActor-isolated, and run-ownership is handled via `Task.isCancelled`
  guards.
- **`MainActor.assumeIsolated` in the Carbon hotkey callback is unsafe** —
  by design: handler installed on `GetApplicationEventTarget()` fires on the
  main run loop (documented in HotKeyCenter.swift:69-70); `assumeIsolated`
  traps rather than corrupts if that ever changes.
- **ToastPresenter leaks panels** (`isReleasedWhenClosed = false`) — false:
  that flag is the correct ARC pattern for NSPanel; the panel deallocates when
  the dismissal Task's strong reference ends after `close()`.
- **Keychain key readable by any process** — false on the facts: generic
  password items are ACL'd to the creating app; other apps trigger a user
  prompt. `FAL_KEY` env fallback is documented, intentional dev convenience.
- **App names sent to fal.ai in prompts** — by design; the prompt needs the
  app context, the user explicitly opted into a third-party generation API.
- **Validate the user's configured terminal command for dangerous patterns**
  — by design: executing the user's own command is the feature.
- **`..` path traversal in project names** — effectively blocked: validation
  rejects `/` and `:`, and a bare `..`/`.` fails the "already exists" check
  (`NewProjectPromptView.swift:86-92`) since the parent always exists.
- **Quoting helpers vulnerable to injection** — checked character-by-character:
  `shellQuoted` uses the standard POSIX `'\''` escape; `appleScriptQuoted`
  escapes `\` and `"`, which is complete for AppleScript string literals.
  (They're untested, which plan 001 fixes — but no bug was found.)
- **PNG polyglot/EXIF risk in downloaded AI images** — mitigated structurally:
  every AI result is re-rendered through a fresh `CGContext`
  (`AIIconStylist.composedIcon`) before being persisted or applied, which
  strips foreign chunks/metadata.
- **TOCTOU between Dock read and icon apply** — accepted: per-app failures
  are collected and reported ("Failed: …"), `DockIconApplier.apply` verifies
  the `Icon\r` resource and rolls back the single app on failure.
- **No rollback for partial Apply-to-Dock runs** — accepted as correct
  semantics: each app's icon is independent; successes are kept, failures are
  named in the status message, and Restore Originals tracks exactly the
  applied set (`styledPaths`).
- **Missing cancellation checks inside AIIconStylist helpers** — immaterial:
  the helpers run millisecond-scale pixel loops on ≤512 px buffers;
  cancellation is checked at the stage boundaries and `URLSession`'s async
  calls are cancellation-aware.
- **`median` returns upper-median for even counts** — immaterial bias in
  backdrop estimation; not worth a change.
- **README lock-badge claim is wrong** — false: `DockAppCell.swift:38-43`
  renders the lock badge for non-customizable apps, plus dimming and a help
  string. README is accurate.
- **Icon collections feature "unintegrated"** — false: `IconCollectionsButton`
  is wired into `DockPaletteControls.swift:36`; the feature is complete in the
  working tree, just uncommitted (see "Before executing" note above).
- **Theme.swift junk drawer / magic-number sweep / FalClient–AIIconStylist
  boundary** — not worth doing at this codebase size; the boundary (transport
  vs. orchestration) is actually sound, and `falModelId` already has a
  centralized, documented override (`defaults write … falModelId`).
- **NSAppleScript/Carbon deprecation risk** — no announced removal timeline;
  monitoring note belongs in CLAUDE.md (plan 005's "sharp edges" covers the
  hotkey/AppleScript context). No action.
- **Preview sources loaded at 256 px upfront in DockReader** — by design:
  previews need them immediately and repeatedly; memory cost is modest.
- **`reload()` re-reads GeneratedIconStore per app** — cold path (explicit
  reload), by design.
